IMU Orientation Viewer

A real-time tool for visualizing, debugging, and calibrating inertial measurement unit (IMU) data directly from your hardware over USB serial. Built for engineers, students, researchers, and hobbyists working with gyroscopes, accelerometers, and magnetometers — connect your sensor, choose a fusion algorithm, and watch your board's orientation come to life in 3D. 🔗 Firmware 📖 Algorithm article

▶ Getting Started
  1. Open the app in Chrome or Edge — Web Serial API is not supported in Firefox or Safari.
  2. Set the Baud rate to match your firmware (default 115200).
  3. Set the Gyro range to match your sensor's full-scale setting.
  4. Click Connect and select your device's COM port from the browser popup.
  5. Choose a Filter: start with Complementary for simplicity, switch to EKF for best accuracy.
  6. Click Reset at any time to zero the yaw and restart the filter state.

No hardware? Click Demo Mode to run a simulated IMU with gyro drift and noise.

📌 Data Format

The viewer expects a 20-byte binary packet per reading:

0xAA  0xFF  [ax_L ax_H]  [ay_L ay_H]  [az_L az_H]
            [gx_L gx_H]  [gy_L gy_H]  [gz_L gz_H]
            [mx_L mx_H]  [my_L my_H]  [mz_L mz_H]
  • Sync bytes: 0xAA 0xFF — marks the start of every packet
  • Values: signed int16, little-endian (LSB first)
  • Accelerometer: raw LSB — scale applied in firmware
  • Gyroscope: raw LSB — set the Gyro range selector to match your FS config
  • Magnetometer: raw LSB — send 0x0000 per axis if no magnetometer is present
🧲 Magnetometer Calibration
  1. Click the Mag Cal button to open the calibration panel.
  2. Slowly rotate the sensor through all orientations — trace full figure-8 patterns in the air.
  3. Watch the scatter plots: each axis-pair should form a circle centred at the origin.
  4. The computed hard-iron bias is shown below each plot — apply it in your firmware to correct raw readings.
  5. Click Clear to reset and start a new calibration session.
